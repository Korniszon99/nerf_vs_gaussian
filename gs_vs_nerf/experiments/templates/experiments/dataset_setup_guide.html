Poradnik przygotowania datasetu

Krok po kroku: od plików na dysku do gotowego datasetu w systemie.

Powrót do dashboardu
Nowy workflow: Wybierz folder z zdjęciami → system automatycznie importuje zdjęcia i przypisuje indeksy.

Krok 1: Przygotuj strukturę folderów

Zalecany layout (COLMAP):

gs_vs_nerf/
  data/
    ilza/
      images/
        IMG_0001.jpg
        IMG_0002.jpg
        ...
      sparse/
        0/
          cameras.bin
          images.bin
          points3D.bin

Krok 2: Uporzadkuj nazwy zdjęć

  • Uzywaj prostych nazw bez spacji i bez znaków specjalnych (np. IMG_0001.jpg).
  • Preferuj .jpg lub .png (stabilniejszy pipeline niż .tif).
  • Nazwy powinny być spójne z rekonstrukcją COLMAP w images.bin.

Krok 3: Dodaj dataset w aplikacji (uproszczony widok)

  1. Kliknij przycisk „Nowy dataset" na dashboardzie.
  2. Wpisz nazwę, np. Ilza.
  3. Wpisz ścieżkę do folderu zawierającego zdjęcia:
    C:\Users\User\PycharmProjects\nerf_vs_gaussian\gs_vs_nerf\data\ilza
    (System szuka images/ wewnątrz folderu, a jeśli nie ma, to przeszukuje sam folder.)
  4. Kliknij „Zapisz".
  5. System automatycznie:
    • Skanuje folder w poszukiwaniu zdjęć (.jpg, .png, .tif, itd.)
    • Sortuje nazwy naturalnie (IMG_2 przed IMG_10)
    • Przypisuje każdemu zdjęciu rosnący indeks (0, 1, 2, 3...)
    • Importuje wszystko w jednym kroku

Krok 4: Weryfikacja i dodatkowe akcje

Po zapisie datasetu zobaczysz:

  • Komunikat o liczbie zaimportowanych zdjęć.
  • Tabelę ze wszystkimi zdjęciami i ich indeksami.

Opcjonalnie: jeśli dodam nowe zdjęcia do folderu, mogę kliknąć przycisk „Re-importuj zdjęcia z folderu", aby dodać je bez duplikatów.

Krok 5: Utworz run eksperymentu

Gdy masz dataset z zdjęciami:

  1. Kliknij „Nowy run" na dashboardzie.
  2. Wybierz dataset, pipeline (vanilla-nerf lub vanilla-gaussian-splatting).
  3. Opcjonalnie podaj config (np. liczba iteracji, skalowanie).
  4. Kliknij „Utwórz i uruchom".

Krok 6: Monitoruj wynik

  • Status runa: pending → running → success/failed.
  • Po zakończeniu sprawdź metryki (psnr, ssim, lpips, duration_sec) i artefakty.
  • Viewer 3D prezentuje chmury punktów .ply.

Najczęstsze problemy

  • „Ścieżka nie istnieje": sprawdź, czy wpisana ścieżka na dysku faktycznie istnieje.
  • „Brak zdjęć w katalogu": system szuka images/ lub plików w głównym folderze. Upewnij się, że są tam zdjęcia.
  • „Brak ns-train": ustaw zmienną środowiska NERFSTUDIO_BIN.
  • Run failuje zaraz po starcie: sprawdź czy data_path zgadza się z layoutem COLMAP i czy nazwy plików są poprawne.
